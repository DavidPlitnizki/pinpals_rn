# Pinpals — Product Strategy & Roadmap

> Этот документ — источник правды для всех решений по продукту.
> Claude должен опираться на него при реализации любых фич.

---

## 🎯 Продуктовая идея

**Pinpals — это не карта мест, это карта твоей жизни.**

Отличие от Apple Maps в одной фразе:
- Apple Maps отвечает на вопрос **«куда мне ехать?»**
- Pinpals отвечает на вопрос **«что значат для меня эти места?»**

Три кита дифференциации:
1. **Memory Layer** — места как эмоциональные воспоминания, а не адреса
2. **Meeting Negotiation** — координация встреч внутри приложения, а не в Telegram
3. **Taste Graph** — персональный граф предпочтений на основе твоих пинов (Phase 2+)


## 📱 Экраны — что менять

### 1. Map (Карта)

**Сейчас**: Карта с пинами, стандартные аннотации.

**Стать**: Живая карта твоей жизни.

Изменения:
- **Пины окрашены по настроению** (mood color из воспоминания), а не по категории
- **Кластеры пинов** показывают превью фото (не число), при тапе — mini-галерея
- **Фильтры**: по настроению / по периоду / по людям ("показать места, где был с Машей")
- **"Год назад"** — кнопка-тригер, которая подсвечивает пины с визитами год назад
- **Режим "Маршрут воспоминаний"** — выбрать несколько пинов, построить маршрут

Убрать: рейтинговые звёзды на карте (это Apple Maps, не мы)

---

### 2. Explore (Исследование)

**Сейчас**: Поиск мест (дублирует Apple Maps).

**Стать**: Открытие мест, похожих на твои любимые.

Изменения:
- **Главный экран**: не поисковая строка, а раздел "Похоже на то, что ты любишь" (на основе твоих тегов и категорий)
- **Карточки мест** с кнопкой "Добавить в мои места" — без отзывов, без рейтингов чужих
- **Поиск** остаётся, но вторичен (в шапке, не в центре)
- **Секция "Исследуй рядом"** — места в радиусе, которых ты ещё не отметил

Убрать: чужие отзывы, агрегированные рейтинги (это Yelp/Google Maps)

---

### 3. My Places (Мои места)

**Сейчас**: Список сохранённых мест.

**Стать**: Личный атлас с фильтрами и воспоминаниями.

Изменения:
- **Три вида**: список / сетка фото / мини-карта (toggle в шапке)
- **Фильтры**: по категории, по тегу, по настроению, по людям, по периоду
- **"Воспоминание дня"** — каждый день в шапке одно место из прошлого (как Google Photos)
- **Статистика**: сколько мест, любимая категория, самый активный месяц (маленький инфо-виджет)
- **Теги как навигация**: облако тегов вместо/рядом с категориями

---

### 4. Place Detail (Детали места)

**Сейчас**: Детали + заметки + фото.

**Стать**: Страница воспоминаний об этом месте.

Изменения:
- **Шапка**: фото на весь экран, поверх — название и mood color
- **Timeline визитов**: вертикальная лента воспоминаний по датам (не просто список заметок)
- **"Добавить воспоминание"** вместо "Добавить заметку" — открывает экран с mood picker, фото, companions
- **Companions**: маленькие аватары/имена тех, с кем был (тапнул — встречи с ними в этом месте)
- **Теги**: редактируемые inline, стиль chip
- **Кнопка "Предложить встречу здесь"** — быстрый переход в CreateMeeting с этим местом

Убрать: рейтинг (звёзды) как основную метрику — заменить на личные теги

---

### 5. Create Memory (новый экран, бывший PlaceNote)

**Это новый экран**, отдельный от просто "добавить заметку".

Шаги (step-by-step sheet):
1. **Фото** — до 5 фото из галереи или камера
2. **Настроение** — горизонтальный скролл emoji/цветов (MemoryMood)
3. **С кем** — ввод имён (+ кнопка из контактов в Phase 2)
4. **Заметка** — свободный текст, опционально
5. **Дата** — по умолчанию сегодня, можно изменить (для ретро-воспоминаний)

---

### 6. CreateMeeting (Создание встречи)

**Сейчас**: Форма с датой и локацией.

**Стать**: Переговорный процесс.

Изменения:
- **Шаг 1**: Название встречи + дата/диапазон
- **Шаг 2**: Предложить места (1-3 варианта из My Places или поиск)
- **Шаг 3**: Участники (Phase 1 — просто имена/контакты, Phase 2 — пользователи)
- **Шаг 4**: Отправить "приглашение" (Phase 1 — шаринг ссылки/текста через Share Sheet)
- **Статус встречи**: draft → proposed → confirmed → done
- **На карте**: показывать маршруты от каждого участника до кандидатов (Phase 2)

---

### 7. Profile (Профиль)

**Сейчас**: Стандартный профиль.

**Стать**: Личная статистика и вкусовая карта.

Изменения:
- **Карта вкусов**: визуализация категорий/настроений в виде bubble chart или bar
- **"Мои паттерны"**: "Ты часто бываешь в тихих местах по утрам", "Любишь природу с друзьями"
- **Счётчики**: мест, воспоминаний, встреч, уникальных городов
- **Любимые теги** — облако
- **Год в местах** — свёрнутый timeline (как Spotify Wrapped для мест)

Предложение на Phase 1:
- **Map** — без изменений, главный таб
- **My Places** — переименовать в **"Мои места"** (оставить)

Иконки должны отражать эмоции, не только функции.

## ✅ Принципы, которым должен следовать Claude при реализации

1. **Никаких чужих отзывов и агрегированных рейтингов** — это делает нас Apple Maps
2. **Mood/emotion first** — каждое место должно иметь эмоциональный контекст
3. **Личное > Публичное** — Phase 1 полностью офлайн/локальный
4. **SwiftData без внешних зависимостей** — строго нативный стек
5. **Design tokens везде** — `Color.Pin.*`, `Font.Pin.*`, `PinSpacing.*`
6. **ViewModels не импортируют SwiftUI** — строго MVVM
7. **Каждый экран = папка** с `Screen.swift` + `ViewModel.swift`
8. **Миграции SwiftData** — при изменении моделей всегда добавлять `VersionedSchema`
